宏观：透过 PromptX 看 AI Coding 市场的演变

第一幕：Vibe Coding 诞生

2025年5月-6月

🚀 故事起点
2025年5月初
• PromptX 只是一堆 .md 提示词文件
• 项目膨胀，提示词越来越多
• 管理成本急剧上升
💡 转折点
痛点：提示词需要代码化管理
决策：让 AI 全权负责代码实现
🎯 v0.1.0 发布（2025.7.9）
从提示词到完整产品
• AI 自主选择：JavaScript
• AI 设计架构：CLI + 资源管理
• AI 实现功能：记忆系统、角色系统、DPML 协议

市场洞察：Vibe Coding 元年

2025年上半年，AI Coding 工具爆发

代表性工具

  • Cursor：Vibe Coding 的代表，对话式编程
  • Claude Code：自然语言驱动开发
  • GitHub Copilot：从补全到生成

关键转变

• 从 "AI 补全代码" → "AI 写代码"
• 从 "提示词技巧" → "意图表达"
• 开发者角色：写代码 → 提需求

PromptX 搭上了 Vibe Coding 的快车

第二幕：遭遇瓶颈

2025年7月-8月 · v0.1.0 - v0.2.0

💣

危机 1

代码量 + 复杂度爆炸
单体架构撑不住
需要工程化、模块化

💣

危机 2

AI 没有统一标准
代码风格不一致
维护成本急剧上升

💣

危机 3

缺少测试覆盖
新增功能怕破坏原有逻辑
严重阻碍迭代速度

用户增多，功能需求爆炸 → PromptX 开始遭遇成长的烦恼

市场洞察：Vibe Coding 的痛苦期

整个行业都在经历类似问题

行业现象

  • Cursor 用户反馈：生成代码质量不稳定
  • GitHub Copilot：大项目中容易"翻车"
  • 社区讨论：需要 "AI + 人类" 的协作规范

关键认知

• Vibe Coding ≠ 银弹
• 复杂项目需要工程化约束
• 测试和质量保证不可或缺

PromptX 的选择：正面应对，引入规范化

第三幕：Spec Coding 重生

2025年9月-至今 · v1.20.0+

🔧 技术重构

• TypeScript 重写（类型安全）
• 模块化架构（pnpm monorepo）
• 完整测试覆盖（单元 + 集成）
• MCP 协议深度集成

📋 Spec Coding 实践

• CCPM：代码规范管理
• Spec-Kit：规范工具包
• Cursor Rules：AI 行为约束
• Claude Code Subagent：专业分工

编程方式的跃迁

从 "完全放手" → "规范约束"
定义标准，AI 按规范执行
人类提供规范和架构，AI 负责实现和优化

市场洞察：Spec + Multi-Agent 时代

2025年下半年，AI Coding 进入新阶段

市场趋势

  • Claude Projects：上下文管理 + 多角色
  • Cursor Rules：规范化配置兴起
  • Agent 框架爆发：Langchain、AutoGPT、CrewAI

关键转变

• 从 "单 AI 生成" → "多 Agent 协作"
• 从 "自由发挥" → "规范约束"
• 从 "个人工具" → "团队平台"

PromptX 的定位

不是替代 Cursor/Claude Code
而是为它们注入专业能力
让每个 AI 都能成为领域专家

PromptX 的三次蜕变

从提示词到 AI 协作平台

阶段 时间 版本 核心特征 AI Coding 模式
诞生 2025.5-7 v0.1.0 从提示词到代码 Vibe Coding
危机 2025.7-8 v0.1.0-v0.2.0 三大瓶颈爆发 痛苦期
重生 2025.9+ v1.20.0+ 规范化 + Multi-Agent Spec + Multi-Agent

关键洞察

从 PromptX 成长史中学到的

1️⃣ Vibe Coding 不是终点

适合快速原型和简单项目
复杂系统需要规范和工程化

2️⃣ Spec Coding 是必经之路

规范 = 人类的经验固化
AI 按规范 = 质量保证
测试覆盖 = 迭代信心

3️⃣ Multi-Agent 是未来

单 AI 有认知边界
多 Agent 分工 = 专业化 + 协作
人类角色：从写代码 → 编排团队

4️⃣ AI Coding 的本质演进

Vibe → Spec → Multi-Agent
= 从助手 → 伙伴 → 团队

给开发者的建议

根据你的阶段，选择正确的方向

如果你现在还在 Vibe Coding
• 享受速度，但要警惕质量
• 小项目可以，大项目需要规范
如果你遇到了瓶颈
• 不是 AI 的问题，是缺少规范
• 引入 Spec：代码规范、测试覆盖、架构约束
如果你想进入 Multi-Agent
• 从角色分工开始
• 用 PromptX 这样的平台赋能
• 人类掌握战略，AI 负责战术
